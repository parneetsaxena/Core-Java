Java Collection Framework (JCF) provides ready-made data structures to use for different cases.


The hierarchy of the framework is as followed:
       I = Interface, C = class
                                                Iterable (I)                                                             Map (I)
                                                    |                                                                    |     |
                        -----------------------Collection (I)---------------------                                       |     Hashtable (C)
                        |                            |                            |                                      |     LinkedHashMap (C)
                     List (I)                    Queue (I)                      Set (I)                                  |     HashMap (C)
                        |                           |   |                        |    |                                  |
                        |                           |  PriorityQueue (C)         |   HashSet (C)                         SortedMap (I)
                     ArrayList (C)              Deque (I)                        |   LinkedHashSet (C)                   |
                     LinkedList (C)----------------|                             |                                       TreeMap (C)
                     Vector (C)               ArrayDeque (C)                SortedSet (I)
                        |                                                         |
                     Stack (C)                                                TreeSet (C)



# Iterable is the Parent Interface. Every class and interface extends or implements it.

- Collection is the interface that extends Iterable and is implemented by List, Map and Set.

# List - List is an interface and it stores data in an orderly manner. It is an ordered collection. Duplicates are allowed and there is indexing for each element.
List is implemented by four classes - ArrayList, LinkedList, Vector and Stack which extends Vector.

ArrayList -> It is a resizable array. It follows the insertion order and the size is dynamic. Internally, it uses an array and when the array is full, it creates a new array of 1.5 times the size of old one and copies all the element in the new one.

LinkedList -> It is a doubly linked list. It works using nodes. Each node stores the address of previous node, data, address of next node.

# Set - Set is a collection which stores unique elements only. Duplicates are not allowed. Maintaining the order is not necessary. 

HashSet -> HashSet uses an array of buckets to store elements. The elements are hashed and stored into buckets. If an element repeats, it matches with the buckets quickly. It uses a hashtable.

LinkedHashSet -> LinkedHashSet works just like HashSet but it uses a linked chain to remember insertion order using LinkedList.

TreeSet -> TreeSet is used to store elements in a sorted tree structure. It uses a self-balancing Binary Search Tree (Red-Black Tree)
